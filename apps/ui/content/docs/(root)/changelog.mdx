## Apr 14, 2026

### OTP Field

The **`input-otp.tsx`** registry component has been removed in favor of **`otp-field.tsx`** (`@coss/otp-field`), which wraps Base UI OTP Field. See the [OTP Field](/ui/docs/components/otp-field) docs and [migration guide](/ui/docs/radix-migration#otp-field) for API details.

**Migration:**

```diff
- npx shadcn@latest add @coss/input-otp
+ npx shadcn@latest add @coss/otp-field
```

```diff
- import { ... } from "@/components/ui/input-otp"
+ import { ... } from "@/components/ui/otp-field"
```

**Agent migration prompt:**

```text
Replace the removed @coss/input-otp / components/ui/input-otp.tsx with @coss/otp-field / components/ui/otp-field.tsx. Follow the OTP Field docs: use OTPField, OTPFieldInput, and OTPFieldSeparator; use length on the root; remove the old input-otp npm package if present.
```

## Apr 12, 2026

### Table

`Table` now supports an optional **`variant`** prop: **`"default"`** (the default) or **`"card"`**. Omitting it keeps the same appearance as before. Use **`variant="card"`** for the card-style grid (separated rows, rounded cells, updated hover/footer treatment)—including tables inside a [Frame](/ui/docs/components/frame) or [CardFrame](/ui/docs/components/card) as shown in the [Table docs](/ui/docs/components/table).

**Migration:**

No change required for existing tables. Opt in where you want the card-style table:

```tsx
<Table variant="card">
```

**Agent migration prompt:**

```text
After upgrading @coss/table, Table supports an optional variant prop: "default" (default) or "card". Existing Table components require no edits. Add variant="card" to Table where you want the card-style table (e.g. inside Frame or CardFrame). Do not add variant to every table—only where the design calls for the card-style grid.
```

## Mar 20, 2026

### Toggle Group

`Toggle` was renamed to `ToggleGroupItem` in `toggle-group.tsx` to avoid a naming conflict with the standalone `Toggle` export.

**Migration:**

```diff
- import { Toggle, ToggleGroup } from "@/components/ui/toggle-group"
+ import { ToggleGroup, ToggleGroupItem } from "@/components/ui/toggle-group"
```

```diff
  <ToggleGroup>
-   <Toggle value="bold">Bold</Toggle>
+   <ToggleGroupItem value="bold">Bold</ToggleGroupItem>
  </ToggleGroup>
```

**Agent migration prompt:**

```text
In my codebase, rename all usages of `Toggle` imported from `toggle-group` to `ToggleGroupItem`. Update both the import statements and JSX usage. Do not change `Toggle` imports from `toggle.tsx` (standalone toggle). Only change files that import `Toggle` from the `toggle-group` module.
```
